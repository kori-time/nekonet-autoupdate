## [0.2]

### Added

- High-availability coordinator architecture
- Sequential fleet updates and deferred reboots
- Canary update support
- Dynamic fleet management
- Service-aware health verification
- Package inventory and maintenance history
- MySQL, PostgreSQL, SQLite, and JSON storage fallback
- Automatic storage recovery and backfill
- Centralized certificate synchronization from `princess.kori.cat`
- Dedicated `nekonet` service account
- Read-only REST API and WebSocket integration for NekoMusic
- Prometheus metrics and Discord notifications

### Changed

- Project documentation now separates version history from the README.

- State machine
- Checkpoint generation and checksum support
- REST API
- WebSocket endpoint
- Prometheus metrics
- Discord notifications
- Storage-provider foundation
- Automated-test foundation
